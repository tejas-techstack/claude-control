# Creative Spin (verified novelty)

Original means checked, not just imagined. Read `../_shared/GUARDRAILS.md` first.

## Workflow
1. **Map the known** — quick /github-explore + web search of the idea space: what exists, what everything shares (the "genre defaults"). You cannot deviate from a norm you have not named.
2. **Force divergence** with mechanical moves applied to the genre defaults:
   - Invert an assumption (what if it worked offline / backwards / for the opposite user?)
   - Transfer a mechanic from a foreign domain (git for recipes, ELO for chores, speedrun timers for study).
   - Change the constraint that shapes everything (1 button only, zero server, must run on a 2010 laptop).
   - Merge two tools nobody merges; or subtract the feature every competitor treats as mandatory.
   Generate 6-8 raw spins fast; do not self-censor yet.
3. **Verify novelty** — search the 3 most promising spins specifically. Finding prior art is fine: state how the spin differs or iterate again. Never claim "no one has done this" without having searched — report what the search showed.
4. **Reality-check the winner**: feasible with free tools? A weekend-sized wedge exists? What is the honest hard part?

## Output format
Top 3 spins, each: the spin in one sentence -> why the genre never does this -> nearest existing thing found (link) and the difference -> feasibility note. Then: "My pick + the 30-minute first experiment." Offer /cross-check to spec it.

## Chaining
- Input: an idea or repo (pairs with /research for academic prior art). Output: a differentiated concept; feeds /cross-check, /design, /agent-automation.
